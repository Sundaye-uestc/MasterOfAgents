# 群聊（多 Agent 协作）测试方案

**测试日期：** 2026-06-04

---

## 一、群聊创建

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 点击 "+ 新建对话" | 弹出创建弹窗 |
| 2 | 选择 "群聊" 类型 | 出现 Agent 选择区域 |
| 3 | 选择 2-3 个 Agent（如 Claude Code + Codex） | Agent 已加入 |
| 4 | 填写群聊标题 "测试群聊" | 标题已输入 |
| 5 | 点击确认创建 | 群聊出现在侧边栏，类型显示"群聊" |

---

## 二、@mention 提及功能

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在输入框输入 `@` | 弹出 Agent 选择列表 |
| 2 | 用 ↑↓ 键选择目标 Agent | 高亮跟随移动 |
| 3 | 按 Enter 选择 | Agent 名称插入，格式 `@AgentName ` |
| 4 | 发送消息 "@Agent1 帮我写一个函数" | 消息发送成功 |
| 5 | 验证只有被 @ 的 Agent 回复 | 仅目标 Agent 响应 |

---

## 三、多 Agent 并行协作

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 发送 "@Agent1 写一个 index.html" | Agent1 创建文件 |
| 2 | 发送 "@Agent2 写一个 style.css" | Agent2 创建文件 |
| 3 | 查看工作区 → 文件 Tab | 两个文件都存在 |
| 4 | 查看工作区 → 变更 Tab | 两次变更分属不同的 Agent |
| 5 | 查看聊天消息下方 InlineDiffCard | 每条消息只显示自己 Agent 的变更（无跨 Agent 冗余） |

---

## 四、跨 Agent 去重验证（4.4 遗留 Bug）

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | Agent1 创建 test.html | 消息下方出现 WebPreviewCard |
| 2 | Agent2 不操作该文件 | Agent2 消息下方不应出现 test.html 的预览卡片 |
| 3 | Agent2 创建自己的 app.js | Agent2 消息下方出现 TextPreviewCard（仅 app.js） |
| 4 | 检查 Agent1 消息下方 | 不出现 app.js（仅 test.html） |

## 五、Orchestrator 编排流程

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在群聊发送复杂任务（不提 @） | Orchestrator 自动分析 |
| 2 | 等待 OrchestratorStatusBar 出现 | 显示任务分解和进度 |
| 3 | 点击展开 Orchestrator 栏 | 查看各 Agent 分配的子任务 |
| 4 | 等待所有 Agent 完成 | 状态栏显示完成 |
| 5 | 查看最终结果 | 所有子任务结果汇总 |

## 六、群聊消息布局

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 不同 Agent 发送消息 | 每条 Agent 消息左侧显示对应的 AgentBadge |
| 2 | 检查 AgentBadge 颜色 | 不同适配器颜色不同（蓝/绿/紫/黄） |

| 4 | 用户消息 | 右对齐，蓝色气泡 |
| 5 | 操作栏 | 每条消息底部始终可见操作按钮 |
